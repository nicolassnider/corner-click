# Corner Click - Web Judges Application

This is the **Corner Click Judges Application** (`apps/web-judges`), a mobile-first, low-latency web application designed for corner sparring referees (judges) to record scores in real-time during ITF Taekwondo matches.

## Core Features

- **Zero-Latency Touch Inputs:** Built strictly under a mobile-first design, featuring native CSS `:active` styling, hardware-accelerated animations, and instant input registration with 0ms click delay.
- **Frictionless Login via PIN:** Judges authenticate instantly using temporary PIN codes generated by Organizers, mapping them directly to their assigned ring Area and Corner role (e.g. Area 1, Corner 1).
- **Offline / Local Mode Bypass (PIN 9999):** Entering PIN `9999` starts a mock local-only scoring session. This session works entirely in memory, bypassing Firebase network calls and server dependencies—excellent for offline arenas or testing.
- **Live Real-time Sychronization:** Syncs warning counts, deductions, and scores in real-time to Firebase Realtime Database (RTDB) to feed the Spectator TV Scoreboard and Jury Dashboard.
- **Strict Viewport Lock:** The layout is locked to `h-[100dvh]` and prevents browser zooming, document text selection, and native "rubber-band" overscrolling on iOS Safari and Chrome.

## Technical Architecture

- **Astro:** Routing wrapper and fast static shell delivery.
- **React:** Powers the login controller (`JudgeApp.tsx`) and the scoring interface (`ScorePad.tsx`).
- **Firebase Auth:** Custom token authentication.
- **Firebase Firestore:** Listens to the judge document at `/tournaments/{tournamentId}/judges/{judgeId}` to receive live assignment data from the Organizer.
- **Firebase Realtime Database:** Syncs live score updates to `/live_matches/{matchId}/scores/{cornerId}`.

## Development

Run the development server from the monorepo root:

```bash
npm run dev --workspace=apps-web-judges
```

Or run dev on all workspaces at once:

```bash
npm run dev
```
